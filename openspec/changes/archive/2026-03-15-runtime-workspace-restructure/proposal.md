## Why

Materialized Claude Code files (`.mcp.json`, `.claude/`, `AGENTS.md`, `skills/`) are currently written to `workspace/` and COPY'd into the Docker image at `/home/mason/workspace/`, but Claude Code resolves these files relative to the project CWD at `/home/mason/workspace/project/` — so they are never found. Additionally, `.chapter/acp.json` is generated by all materializers but has no consumer anywhere in the codebase.

## What Changes

- **BREAKING**: Materializer output is split into two destinations:
  - `build/workspace/project/` — files that must appear in the project directory (`.mcp.json`, `.claude/`, `AGENTS.md`, `skills/`)
  - `workspace/` — files consumed by `agent-entry` at bootstrap (`agent-launch.json`)
- **BREAKING**: The Dockerfile `COPY workspace/` directive is replaced by `COPY build/workspace/` — workspace is no longer baked into the image
- `workspace/` becomes a live bind mount to `/home/mason/workspace/` at runtime (same pattern as `home/`)
- Files from `build/workspace/project/` are mounted as individual bind overrides onto `/home/mason/workspace/project/` at runtime, using the same overlay mechanism as `container.ignore.paths`
- **BREAKING**: `.chapter/acp.json` is removed from all materializer outputs
- `generateAcpConfigJson` is removed from `agent-sdk` and all re-exports

### Modified Capabilities

- `claude-code-materializer`: Output paths for `.mcp.json`, `.claude/`, `AGENTS.md`, `skills/` move to project-relative keys; `.chapter/acp.json` output removed entirely
- `agent-dockerfile`: `COPY` directive targets `build/workspace/` instead of `workspace/`; workspace directory is no longer embedded in the image
- `docker-compose-generation`: Agent service gains a `workspace/` bind mount to `/home/mason/workspace/`; `build/workspace/project/` files are enumerated as individual bind mounts overlaid on the project mount
- `agent-sdk`: `generateAcpConfigJson` function and its export are removed
