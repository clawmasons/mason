# Design: CLAWMASONS_HOME Utility & chapters.json

**Date:** 2026-03-10

## Approach

A single module at `packages/cli/src/runtime/home.ts` provides all `CLAWMASONS_HOME` resolution and `chapters.json` CRUD operations. All functions are pure (take `home` path as parameter) except `getClawmasonsHome()` which reads the environment.

### Key Design Decisions

1. **Dependency Injection for Home Path** -- All functions except `getClawmasonsHome()` accept the home directory as a parameter rather than reading from the environment. This makes testing straightforward (use temp directories) and allows callers to override the path.

2. **Upsert Semantics** -- `upsertRoleEntry()` matches by the composite key `(lodge, chapter, role)`. If found, it updates the existing entry's fields and sets `updatedAt`. If not found, it appends a new entry with both `createdAt` and `updatedAt`.

3. **Atomic Writes** -- `writeChaptersJson()` writes to a temp file then renames, preventing corruption from interrupted writes.

4. **Graceful Missing File** -- `readChaptersJson()` returns `{ chapters: [] }` when the file doesn't exist, so callers never need null checks.

5. **Ensure Home Idempotent** -- `ensureClawmasonsHome()` creates the directory and `.gitignore` only if they don't already exist. Safe to call multiple times.

### Types

```typescript
interface ChapterEntry {
  lodge: string;
  chapter: string;
  role: string;
  dockerBuild: string;
  roleDir: string;
  targetDir?: string;
  agents: string[];
  createdAt: string;
  updatedAt: string;
}

interface ChaptersJson {
  chapters: ChapterEntry[];
}
```

### Function Signatures

```typescript
function getClawmasonsHome(): string;
function ensureClawmasonsHome(home: string): void;
function readChaptersJson(home: string): ChaptersJson;
function writeChaptersJson(home: string, data: ChaptersJson): void;
function findRoleEntry(home: string, lodge: string, chapter: string, role: string): ChapterEntry | undefined;
function upsertRoleEntry(home: string, entry: ChapterEntry): void;
```

### File Layout

```
CLAWMASONS_HOME/
  chapters.json          # Registry of initialized roles
  .gitignore             # Ignores logs/ subdirectories
```

The `.gitignore` content:
```
# Auto-generated by chapter CLI
**/logs/
```

### Error Handling

| Scenario | Behavior |
|----------|----------|
| `chapters.json` missing | `readChaptersJson` returns `{ chapters: [] }` |
| `chapters.json` malformed JSON | Throws with descriptive error |
| `CLAWMASONS_HOME` dir missing | `ensureClawmasonsHome` creates it |
| Write fails (permissions) | Throws native fs error |

### Backward Compatibility

This is a new module with no existing API to maintain. Future changes (init-role, run-agent, run-acp-agent) will import from this module.
