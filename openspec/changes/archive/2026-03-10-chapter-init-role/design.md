# Design: `chapter init-role` Command

**Date:** 2026-03-10

## Approach

A new command module at `packages/cli/src/cli/commands/init-role.ts` that orchestrates host-wide role initialization. Leverages existing utilities from `runtime/home.ts` and `docker-init.ts` for `CLAWMASONS_HOME` management and chapter config reading.

### Key Design Decisions

1. **Reuse `readChapterConfig`** -- The `.clawmasons/chapter.json` in the chapter workspace already contains the `<lodge>.<chapter>` identifier. We parse it to derive `lodge` and `chapter` slugs.

2. **Discover all agents for the role** -- Instead of initializing for a single agent, the command discovers all agent packages in the workspace, resolves them, and finds every agent that defines the specified role. All matching agents get service entries in the compose file.

3. **`${PROJECT_DIR}` substitution** -- The `docker-compose.yaml` uses environment variable references (`${PROJECT_DIR}`) for workspace mount paths. `run-agent` and `run-acp-agent` set `PROJECT_DIR` at runtime. Tokens (`${CHAPTER_PROXY_TOKEN}`, `${CREDENTIAL_PROXY_TOKEN}`) are also env var references, generated fresh per session.

4. **Backup on re-run** -- If `docker-compose.yaml` already exists in the role directory, it is copied to `docker-compose.yaml.bak` before overwriting. This provides a simple rollback mechanism.

5. **Dependency injection** -- All external dependencies (package discovery, agent resolution, chapter config reading, filesystem operations) are injectable via a `deps` parameter for testability.

### CLI Signature

```
chapter init-role --role <name> [--agent <name>] [--target-dir <path>]
```

- `--role <name>` (required): Role to initialize
- `--agent <name>` (optional): Specific agent to include (default: all agents with the role)
- `--target-dir <path>` (optional): Override default role directory location

### Function Signatures

```typescript
interface InitRoleDeps {
  discoverPackagesFn?: (rootDir: string) => Map<string, DiscoveredPackage>;
  resolveAgentFn?: (name: string, packages: Map<string, DiscoveredPackage>) => ResolvedAgent;
  readChapterConfigFn?: (rootDir: string) => ChapterConfig;
}

function registerInitRoleCommand(program: Command): void;

function initRole(
  rootDir: string,
  options: { role: string; agent?: string; targetDir?: string },
  deps?: InitRoleDeps,

function generateInitRoleComposeYml(opts: {

```

### docker-compose.yaml Template

```yaml
# Generated by chapter init-role
services:
  proxy-<role>:
    build:
      context: "<dockerBuildPath>"
      dockerfile: "proxy/<role>/Dockerfile"
    volumes:
      - "${PROJECT_DIR}:/workspace"
      - "./logs:/logs"
    environment:
      - CHAPTER_PROXY_TOKEN=${CHAPTER_PROXY_TOKEN}
      - CREDENTIAL_PROXY_TOKEN=${CREDENTIAL_PROXY_TOKEN}
    restart: "no"

  credential-service:
    build:
      context: "<dockerBuildPath>"
      dockerfile: "credential-service/Dockerfile"
    environment:
      - CREDENTIAL_PROXY_TOKEN=${CREDENTIAL_PROXY_TOKEN}
    depends_on:
      - proxy-<role>
    restart: "no"

  agent-<agent>-<role>:
    build:
      context: "<dockerBuildPath>"
      dockerfile: "agent/<agent>/<role>/Dockerfile"
    volumes:
      - "${PROJECT_DIR}:/workspace"
    depends_on:
      - credential-service
    environment:
      - MCP_PROXY_TOKEN=${CHAPTER_PROXY_TOKEN}
    init: true
    restart: "no"
```

### Directory Structure Created

```
CLAWMASONS_HOME/
  chapters.json                (updated with new entry)
  .gitignore                   (created if missing)
  <lodge>/
    <chapter>/
      <role>/
        docker-compose.yaml    (generated)
        docker-compose.yaml.bak (if re-running)
        logs/                  (created)
```

### Error Handling

| Scenario | Behavior |
|----------|----------|
| No `.clawmasons/chapter.json` in workspace | Throws: "Run chapter init first" |
| Role not found in any agent | Throws: "Role <name> not found" |
| `--agent` specified but doesn't exist | Throws: "Agent <name> not found" |
| Agent exists but doesn't have the role | Throws: "Agent <name> does not have role <role>" |
| Docker build path missing | Throws: "Run chapter build first" |

### Backward Compatibility

This is a new command. The existing `run-init` command is NOT removed by this change (that was done in CHANGE 3). The `init-role` command is additive.
