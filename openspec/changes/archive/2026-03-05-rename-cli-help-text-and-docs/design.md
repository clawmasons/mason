## Design: Rename CLI Help Text, Output Messages, and Documentation

### Approach

This is a string-literal rename change. No structural, API, or behavioral changes. Every user-facing string that says "forge" becomes "chapter", and every internal comment or log prefix is updated to match.

### Source File Changes

#### 1. CLI Framework (`src/cli/index.ts`)
- `.name("forge")` -> `.name("chapter")`
- Description: `"Agent Forge System -- AI agent packaging, governance, and runtime orchestration"` -> `"Clawmasons Chapter -- AI agent packaging, governance, and runtime orchestration"`

#### 2. CLI Commands (`src/cli/commands/`)

**init.ts:**
- `.description("Initialize a new forge workspace")` -> `"Initialize a new chapter workspace"`
- `"forge workspace initialized"` -> `"chapter workspace initialized"`
- All "Next steps" lines: `forge list` -> `chapter list`, `forge validate` -> `chapter validate`, `forge install` -> `chapter install`, `forge add` -> `chapter add`, `forge build` -> `chapter build`
- Comment: `"templates directory inside the forge package"` -> `"templates directory inside the chapter package"`

**add.ts:**
- `.description("Add a forge package dependency ...")` -> `"Add a chapter package dependency ..."`

**proxy.ts:**
- `.description("Start the forge MCP proxy server for an agent")` -> `"Start the chapter MCP proxy server for an agent"`
- `"Shutting down forge proxy..."` -> `"Shutting down chapter proxy..."`
- `"Make sure you're in a forge workspace root"` -> `"Make sure you're in a chapter workspace root"`

**install.ts:**
- `"forge run ${agentName}"` -> `"chapter run ${agentName}"`

**run.ts:**
- `"forge install ${agentName}"` -> `"chapter install ${agentName}"`

**stop.ts:**
- `"forge install ${agentName}"` -> `"chapter install ${agentName}"`

**docker-utils.ts:**
- `"Run forge install first"` -> `"Run chapter install first"`

#### 3. Proxy (`src/proxy/`)

**upstream.ts:**
- Client name `"forge-upstream"` -> `"chapter-upstream"`

**hooks/approval.ts:**
- All `[forge]` log prefixes -> `[chapter]`

**hooks/audit.ts:**
- `[forge]` log prefix -> `[chapter]`

#### 4. Compose (`src/compose/docker-compose.ts`)
- Comment: `"runs \`forge proxy\` natively"` -> `"runs \`chapter proxy\` natively"`

#### 5. Materializer (`src/materializer/`)

**types.ts:**
- Comment: `"abstract forge dependency graph"` -> `"abstract chapter dependency graph"`

**claude-code.ts:**
- `"Generated by forge from"` -> `"Generated by chapter from"`
- `"managed by forge (Agent Forge System)"` -> `"managed by chapter (Clawmasons Chapter)"`
- Comment: `"single \"forge\" MCP server entry"` -> `"single \"chapter\" MCP server entry"`

### README.md

Full rewrite:
- Title: `# chapter` (was `# forge`)
- Description: `Clawmasons Chapter` (was `Agent Forge System`)
- Package names: `@clawmasons/chapter` (was `@clawmasons/forge`)
- CLI commands: `chapter init`, `chapter list`, etc.
- Package references: `@clawmasons/chapter-core` (was `@clawmasons/forge-core`)
- Directory references: `chapter-core/` (was `forge-core/`)
- API imports: from `"@clawmasons/chapter"`
- CLI binary: `bin/chapter.js`
