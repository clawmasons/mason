## Why

The `chapter-members` PRD (REQ-001, REQ-010) requires that all CLI help text, output messages, error messages, and documentation consistently reference "chapter" instead of "forge". Changes #1-#3 renamed the metadata field, npm packages, directories, and environment variables, but user-facing string literals -- help text, console.log output, error messages, comments, and the README -- still contain "forge" references.

The current codebase has ~30+ remaining "forge" string literals across:
- CLI program name and description (`src/cli/index.ts`)
- CLI command descriptions and output messages (`src/cli/commands/*.ts`)
- Proxy log prefixes and shutdown messages (`src/proxy/hooks/*.ts`, `src/cli/commands/proxy.ts`)
- Materializer comments and generated content (`src/materializer/*.ts`)
- Upstream MCP client name (`src/proxy/upstream.ts`)
- README.md (full documentation still references forge)
- Test files with forge assertions and temp directory prefixes
- OpenSpec spec files with forge examples

## What Changes

- `src/cli/index.ts`: `.name("forge")` -> `.name("chapter")`, description updated
- `src/cli/commands/init.ts`: description, success message, "Next steps" output all updated to "chapter"
- `src/cli/commands/add.ts`: description updated
- `src/cli/commands/proxy.ts`: description, shutdown message, error messages updated
- `src/cli/commands/install.ts`: "Next steps" output `forge run` -> `chapter run`
- `src/cli/commands/run.ts`: error messages `forge install` -> `chapter install`
- `src/cli/commands/stop.ts`: error messages `forge install` -> `chapter install`
- `src/cli/commands/docker-utils.ts`: error message `forge install` -> `chapter install`
- `src/proxy/upstream.ts`: client name `"forge-upstream"` -> `"chapter-upstream"`
- `src/proxy/hooks/approval.ts`: log prefix `[forge]` -> `[chapter]`
- `src/proxy/hooks/audit.ts`: log prefix `[forge]` -> `[chapter]`
- `src/materializer/types.ts`: comment "forge dependency graph" -> "chapter dependency graph"
- `src/materializer/claude-code.ts`: "Generated by forge" -> "Generated by chapter", "managed by forge" -> "managed by chapter"
- `src/compose/docker-compose.ts`: comment "runs `forge proxy`" -> "runs `chapter proxy`"
- `README.md`: full rewrite for chapter terminology
- Test files: update assertions and temp directory prefixes
- OpenSpec spec files: update forge references in active specs
